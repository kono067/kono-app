## 5. API エンドポイント

### 認証
| メソッド | パス | 説明 |
|---------|------|------|
| ALL | `/api/auth/*` | Better Auth ハンドラー（ログイン・ログアウト等） |

### 議事録生成
| メソッド | パス | 説明 |
|---------|------|------|
| POST | `/api/generate` | メモ + テンプレートから議事録を生成 |

**リクエスト:**
```json
{
  "memo": "・A社 見積 来週提出\n・UI 修正必要",
  "template": "社内会議"
}
```

**レスポンス:**
```json
{
  "result": "■議題\n...",
  "creditsRemaining": 4
}
```

### 議事録管理
| メソッド | パス | 説明 |
|---------|------|------|
| GET | `/api/minutes` | 議事録一覧取得 |
| GET | `/api/minutes/:id` | 議事録個別取得 |
| DELETE | `/api/minutes/:id` | 議事録削除 |

### クレジット
| メソッド | パス | 説明 |
|---------|------|------|
| GET | `/api/credits` | 残りクレジット取得 |

### 決済
| メソッド | パス | 説明 |
|---------|------|------|
| POST | `/api/checkout` | Polar チェックアウトセッション作成 |
| POST | `/api/webhooks/polar` | Polar Webhook 受信 |

## 6. Gemini API プロンプト設計

### システムプロンプト
```
あなたは会議議事録を作成するアシスタントです。
ユーザーが入力した会議メモを、指定されたテンプレート形式に整理して議事録を作成してください。

ルール：
- メモの内容を正確に反映すること
- メモにない情報を勝手に追加しないこと
- 各セクションに該当する内容がない場合は「特になし」と記載すること
- 敬語・丁寧語で記載すること
- 簡潔でわかりやすい文章にすること
```

### ユーザープロンプト
```
以下のメモを「{テンプレート名}」テンプレートの形式で議事録にしてください。

テンプレート形式：
{テンプレートの各セクション}

メモ：
{ユーザー入力のメモ}
```

## 7. 環境変数

### ローカル開発用 (.env)
```env
# Gemini API
GEMINI_API_KEY=AIzaSy...

# Turso Database
TURSO_DATABASE_URL=libsql://xxx.turso.io
TURSO_AUTH_TOKEN=xxx

# Better Auth
BETTER_AUTH_SECRET=ランダムな文字列
BETTER_AUTH_URL=http://localhost:5173

# Google OAuth
GOOGLE_CLIENT_ID=xxx.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=xxx

# Polar
POLAR_ACCESS_TOKEN=xxx
POLAR_WEBHOOK_SECRET=xxx

# Polar Product IDs
POLAR_PRODUCT_UME=xxx
POLAR_PRODUCT_TAKE=xxx
POLAR_PRODUCT_MATSU=xxx
```

### Vercel 本番環境
上記と同じ変数を Vercel ダッシュボードの Environment Variables に設定する。
`BETTER_AUTH_URL` は本番ドメインに変更する。

## 8. セキュリティ対策

| 対策 | 実装方法 |
|------|---------|
| APIキー保護 | サーバーサイド（Serverless Functions）でのみ使用 |
| 認証 | Better Auth でセッションベース認証 |
| CSRF保護 | Better Auth の組み込み CSRF 保護 |
| Webhook検証 | Polar の署名検証 |
| CORS | Vercel のデフォルト設定（同一オリジン） |
| 入力検証 | メモテキストの長さ制限（最大5000文字） |
